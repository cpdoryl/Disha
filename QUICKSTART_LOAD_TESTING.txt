TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

  Issue: "Cannot access http://localhost:3000"
  ─────────────────────────────────────────────
  Solution: Backend not started yet
  - Verify it with: curl http://localhost:3000/health
  - Or wait 30 seconds after running start-backend-for-testing.ps1
  - Check terminal 1 for startup logs


  Issue: "wrk: command not found"
  ──────────────────────────────
  Solution: Install wrk
  - choco install wrk -y
  - Or download from: https://github.com/wg/wrk/releases


  Issue: "Error rate high during test"
  ────────────────────────────────────
  Check: Database health
  - docker exec -e PGPASSWORD=staging_password_change_me disha-staging-postgres psql -U staging_user -d disha_staging_db -c "SELECT 1;"

  Check: API logs
  - docker logs disha-staging-api --tail 20


  Issue: "Memory continuously growing"
  ────────────────────────────────────
  Possible: Memory leak in application
  Action: Reduce load, monitor with: curl http://localhost:3000/health/metrics
